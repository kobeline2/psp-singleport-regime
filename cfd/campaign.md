# CFD キャンペーン計画

シリーズ = 「Excel の 1 シート」に相当する, 目的を共有するケース群.
個々のケースの行は `cases.csv` が正本. ここには各シリーズの目的・前提・合格基準・決定だけを書く.
シリーズは番号順に消化するが, B (メッシュ作成) は A と並行で準備してよい.

進め方の原則: 変更は 1 ケース 1 種類. 判定基準を先に書く. 結果が出たら「決定」欄を埋めてから次へ.

## シリーズ A — 数値設定の高速化 (現行メッシュ, 物理不変)

**目的**: 答えを変えずに 1 ステップの壁時計を下げ, 「確定設定 v1」を凍結する.

**前提タスク**:
- [x] コア数確認 (2026-07-22 実測): Intel Core Ultra 9 285K = **24 コア (P8 + E16, HT なし)**,
      RAM 248 GB. 16 ランクはオーバーサブスクライブではない. P/E 混成のため最適ランク数は
      別途掃引 (A06). メモリはデュアルチャネル DDR5 なので帯域律速に注意
- [x] p_rgh 反復解析 (2026-07-22, 対象を変更して完了): パイロット log は 7/20 の作り直しで
      消失していたため, 進行中の X05 (1.81M セル, RAS) の 55 MB log を解析.
      **補正別反復 = 3.2 / 14.3 / 1.1, 計 18.6/step** — tank (X01) からの外挿予測 (80-120) より
      はるかに健全. 深掘りは2回目の補正が担い, Final は実質無料という構造.
      → **GAMG の期待値を下方修正: 2-4x ではなく 1.2-1.5x 程度**. A01 の 20 s A/B は
      安いので実測はするが, 優先度は下げる. 6月の tank では最終補正が平均 37.7 だったので
      メッシュ/レジームに依存する — 本番設定を凍結する時点でもう一度測る
- 派生知見: nCorrectors 3→2 と p_rghFinal 緩和の利得も同様に小さい (Final がすでに~1反復).
      現状の X05 の律速は (a) 1.81M セル自体のコスト, (b) 浅水期の Courant 制限 (cap 到達 0.8%),
      (c) RAS の追加コストと bounding. → **最大のレバーはメッシュ (シリーズ B の六面体化) と
      乱流方針の再検討に移った**
- [x] 質量収支診断 (2026-07-22): X05 の流入パッチ phi はちょうど -5.000e-4 だが dV/dt は
      Q の 90-93% で一定 → **水が領域内部ではなく「入口で入り損ねている」**ことが判明.
      経緯: A07 (laminar化) でも 93.1% のまま → RAS はシロ. diff により 7/21 に学生が
      alpha 流入 BC を fixedValue 1 → variableHeightFlowRate に変更していたことを特定.
      **機構**: U 側 BC が面ごとに U ∝ alpha で流量配分するため, 体積流束の合計はちょうど Q だが
      水の流束は Q x (Σα²/Σα) < Q. 界面がパッチを横切る間 (h < 0.055 m) は常に数%欠損する.
      A00 (fixedValue 1) が 99.8% 保存だったことと整合. **A08 で確定: BC を戻すと 100.0% に回復.**
      **トレードオフ (実測)**: fixedValue 1 = 質量厳密だが落下流で dt 8.94→5.64 ms
      (浅水期の壁時計 +62%) / variableHeightFlowRate = dt は大きいが水が 7% 欠損.
      **当面の方針 = fixedValue 1** (水位からレジームを判定する研究なので質量厳密は譲れない).
      恒久策: B01 の六面体メッシュで「常時水没の流入形状」(導水路上流端の下部だけを inlet にする等)
      を設計すれば, 質量厳密と大きな dt を両立できる見込み

**方法**: パイロット (A00) の t = 500 s からのリスタート 20 s 区間で A/B.
比較指標 = ExecutionTime の階差 (s/step). 健全性 = Min/Max(alpha), continuity error, waterVolume の傾き.

**合格基準**: A05 まで消化した時点で A00 比 2.5 倍以上, かつ健全性チェック全て通過.
A04/A05 (dt 緩和) は答えを変えうるので, 表面速度場のスナップショット比較まで行う.

**決定 (2026-07-22 記入)**: **確定設定 v1 = laminar + alpha流入BC fixedValue 1 + nCorrectors 2 + PCG.**
- 根拠: A07 (RASコスト +38% を回避, boundingはRAS固有と確定), A08 (質量 100.0% 回復),
  A09 (nCorrectors 2 で 1.27x, 健全性同一), A10 (GAMG はこの構成では 3% 遅く不採用.
  圧力反復がスパイクする局面の保険として選択肢に残す)
- 同レジームの 1 ステップ単価で X05 比 ~1.7x. ただし fixedValue 1 の滝で浅水期の dt が -37%
  なので, 正味の壁時計は浅水期は X05 とほぼ同等, ポート水没後 (h>55mm) は ~1.7x
- 現メッシュ + v1 での目安: 実時間 1000 s ≈ 40 h 前後 → **高速化の主戦場はシリーズ B に移行**
  (六面体 ~0.75M で単価半減 + 常時水没インレットで滝と dt 制約を同時に解消 + その後 dt 緩和)
- 残タスク: nCorrectors 2 採用の表面速度場スナップショット確認 (本番初回ランで実施),
  A02 renumberMesh (任意, +5-10% 見込み), A06 ランク掃引は B メッシュ上で実施,
  A04/A05 (Courant/maxDeltaT 緩和) も B メッシュ上で実施する (滝が消えてから測るのが正しい)

## シリーズ C — スタート水位・スピンアップ感度

**目的**: 「どの水位から静止で始めるか」を実測で確定する.
実験は 0 mm から充填し 30 mm 到達時 (履歴 ~364 s) に PIV 開始. CFD は 18.33 mm 静止スタートで
履歴 142 s. 循環 1 回転 ~250-300 s に対しどちらも未発達で, CFD が 220 s ぶん浅い.

**前提タスク**:
- [~] 偏向チェック (2026-07-23, 対象を B03 に変更して一次実施): z=20-30mm スラブの左右運動量積分
      13 時刻で測定. 結果: **固定した偏向は存在せず, 蛇行 (符号が O(100s) で反転)**.
      t=120 (h=30mm, PIV開始相当) 時点で流れは既に活発かつ非対称 (|IxL|:|IxR| ≈ 3:1) だが
      方向はロックされない. → C の問いは「偏向確立までの時間」から「蛇行統計の定常化時間」へ
      再定義. 注意: スラブ積分は循環が相殺される鈍い指標. 確定判断は importer 経由の
      表面場メトリクス (E, I_asym, I_unst) で行う
- [x] 表面サンプル解析 (7/23, B03 全201時刻を surfSample で抽出, local/work/B03/surfSample に回収):
      (1) isoSurface の z = V/A の水位と全時刻一致 (計測系の自己整合 ✓)
      (2) 表面平均流速 1.5-3.0 cm/s — ジェット運動量からの事前推定 U_b~2cm/s と整合
      (3) **蛇行を表面でも確認**: 左右非対称指数が ±0.1-0.3 で符号反転を繰り返す.
          後半 (h>80mm, B4) で振幅増大 (-0.2〜-0.3) — 深水側で非対称が強まる傾向の兆候
      (4) mean|U| が非単調: 2.7→1.6 cm/s (B1→B2で減速) → 3.0 cm/s (B4で再加速).
          **E(t) が h/a≈1 付近で極小になる可能性** — 実験バンド比較の注目点
      (5) isoSurface 点数が t≈600 (気泡パージ完了期) と t≈900 (質量ドリフト期) に 3-4 倍スパイク
          = **界面断片化の診断指標を発見**. クリッピング損失の実行犯とみられる.
          → importer に z フィルタ (中央値±15mm) を追加済み (気泡面を PIV 比較から除外)

**方法**: スタート水位 {30, 20, 10} mm で B1 区間 (30→50 mm) を計算し,
B1 の band-mean 場と E, phi_lv, I_asym, I_circ, I_unst を比較.

**合格基準**: スタート水位による差が実験の反復間ばらつき未満 → 安い設定を採用.
差が大きい → 10 mm スタート (発達 243 s) を標準とし, 論文 Methods に明記.
